Google SDE Interview Guide

45+ Questions: Deep-dive into Coding, Systems, and the unique "Googliness" culture fit.

Coding & DSA (Core)

1. Find the shortest path from a starting node to all other nodes in a graph with non-negative weights. Medium
Topic: Graphs (Dijkstra's Algorithm). Key Requirement: Must handle large datasets efficiently.
2. Given a binary search tree, find the $k$-th smallest element in it. Easy
Topic: BST / Trees (Inorder Traversal). Optimization: Can be optimized using morris traversal for O(1) space.
3. Implement a rate limiter to handle requests per second (e.g., 5 requests/second) for a distributed microservice. Hard
Topic: Data Structures / System Design (Sliding Window/Token Bucket). Key Concept: Distributed counter mechanism.
4. Design and implement a LFU (Least Frequently Used) cache. Hard
Topic: Data Structures. Key Concept: Requires maintaining frequency counts and using a doubly linked list or min-heap.

System Design (G-Scale)

1. Design Google Maps: focus on scale, routing, and real-time updates. System Design
Key Concepts: Geo-hashing, Load Balancing, Microservices, and Graph databases for routing.
2. Design a Distributed Web Crawler. System Design
Key Concepts: Frontier management, DNS resolution, Duplication Handling, and Sharding.
3. Explain the Paxos consensus algorithm and where Google uses it (e.g., Spanner). Conceptual
Key Concepts: Distributed Consistency, Leader Election, and Quorum requirements.

HR & Behavioral ('Googliness')

1. Tell me about a time you had to challenge a manager or a senior colleague's decision. Behavioral
Focus: Courage, Disagree and Commit. Answer using the STAR method.
2. What does 'Googliness' mean to you, and how do you embody it? Behavioral
Focus: Culture Fit, Ambiguity, Intellectual Curiosity, and Impact.
3. Describe a project where you failed, and what did you learn from it? Behavioral
Focus: Growth Mindset, Resilience, and ability to take calculated risks.
